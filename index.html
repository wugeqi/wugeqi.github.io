JAVA

1.JAVA基础

（1）jvm、jre、jdk：

jvm：java虚拟机，是程序运行时的环境，可以把字节码转换成机器码，并执行程序

jre:程序运行的的最小环境，包含了jvm和一组库

jdk：java开发工具包

面向对象：将世界上的一切东西都抽象成了一个对象，然后每个对象都有自己的属性和方法，程序功能的实现就是通过对象的交互实现的

继承：指一个类可以共享另一个类中的所有对象和方法

封装：指将对象的属性和行为结合在一起，对象不对外暴露细节，只对外界提供交互的接口

多态：多个类的对象可以对同一个事务做出响应

运行时多态：重写，指子类可以重写父类的方法

编译时多态：重载，指在同一个类中允许参数不同的重名方法

反射：运行时可以获取任意一个类的所有信息，字段，方法等

获取.class：1.根据完整包名2.根据实例的对象获得3.根据包名获得

泛型：允许类、接口和方法在定义时使用一个或多个类型参数，这些类型在使用时可以指定为具体的类型

动态代理：运行时动态生成代理

方法：

jdk：主要是对实现接口的类，通过调用invocationhander，并重写invoke（）方法

cglib：主要是对没有实现接口的类，通过创建子类继承父类，然后对方法进行重写，实现动态代理

注解：一种声明式接口，根据注解决定程序的执行的方式

生命周期：

字节：编译时没有

编译：运行时没有

运行：都有

方法：反射获取字段，然后根据动态代理对原方法进行扩展

深拷贝和浅拷贝：

浅拷贝：不会递归拷贝引用，，也就是说如果是个引用对象，进行浅拷贝的时候，会创建一个新对象存储着原来的引用，也就是说两个对象指向着同一个引用

深拷贝：会递归拷贝引用

修饰符：

final：表示对象不可更改不可继承，如果修饰的是基本数据类型，表示其值一直不变，如果修饰的是引用数据类型，表示执行的引用一直不变，但是引用的对象中的值可以改变

static：静态修饰符，可以修饰变量、方法、类一旦经过修饰，就会变成类的本身一部分。在类加载时被初始化，不依赖类的实例

abstarct:抽象，可以修饰类和方法，抽象类不可以被实例化，抽象方法只有声明没有实现

接口：一个完全抽象的类，定义一组的行为规范，不提供具体实现，只有常量和抽象方法

抽象类：用来描述类的共同特性和行为，可以有成员变量、构造方法和具体方法

设计模式

工厂模式

定义了一个创建对象的抽象方法，由子类决定要实例化的类。工厂方法模式将对象的实例化推迟到子类。

策略模式

策略模式定义了一系列算法，并将每个算法封装起来，使他们可以相互替换，且算法的变化不会影响到使用算法的客户。允许你在运行时选择算法的实现，通过将不同的行为（策略）封装在独立的类中，从而将这些行为抽象出来。这样可以让客户端通过不同的策略对象，选择不同的实现方式，而不需要改变客户端代码。

模板模式

定义一个操作中算法的骨架，而将一些步骤延迟到子类中，模板方法使得子类可以不改变算法的结构即可重定义该算法的某些特定步骤。

完成一件事情，有固定的数个步骤，但是每个步骤根据对象的不同，而实现细节不同；就可以在父类中定义一个完成该事情的总方法，按照完成事件需要的步骤去调用其每个步骤的实现方法。每个步骤的具体实现，由子类完成。

spring

1.spring

定义：一种开发框架

特性：

di:依赖注入是将对象的创建和依赖关系的管理交给 Spring 容器来完成，类只需要声明自己所依赖的对象，容器会在运行时将这些依赖对象注入到类中，从而降低了类与类之间的耦合度，提高了代码的可维护性和可测试性。

实现方式：

构造器注入

setter方法注入

注解：@Autowired

ioc：控制反转，将对对象的控制权交给spring

实现：di和反射

设计模式：工厂模式

基于xml

基于注解

aop：面向切面编程，能够将那些与业务无关，却为业务模块所共同调用的逻辑封装起来，以减少系统的重复代码，降低模块间的耦合度

实现：动态代理：jdk/cglib

应用：日志 事务

bean的生命周期：

  1. 实例化：为 Bean 分配内存空间；
  2. 设置属性：将当前类依赖的 Bean 属性，进行注入和装配；
  3. 初始化：
    1. 执行各种通知；
    2. 执行初始化的前置方法；
    3. 执行初始化方法；
    4. 执行初始化的后置方法。
  4. 使用 Bean：在程序中使用 Bean 对象；
  5. 销毁 Bean：将 Bean 对象进行销毁操作。

循环 依赖：

通过构造方法进行依赖注入时产生的循环依赖问题。（未解决）

通过setter方法进行依赖注入且是在多例（原型）模式下产生的循环依赖问题。（未解决）

基于setter依赖注入且是单例模式下（已解决）

解决方法：三级缓存（通过将对象提前暴露出来）

数据结构：

一级缓存：存放的已经实例化的bean

二级缓存：早期的bean引用，已经实例化但是还未完全初始化的bean

三级缓存：存放的是ObjectFactory对象，这些对象可以生成早期的bean引用。那么这个正在创建的bean就会通过这个ObjectFactory来创建一个早期引用

步骤：

以Bean A和Bean B相互依赖为例：

创建Bean A：
实例化A（调用构造器，此时对象尚未初始化）。
将A的工厂对象（用于生成早期引用）放入三级缓存。
开始填充A的属性，发现需要注入Bean B。

创建Bean B：
实例化B，同样将B的工厂对象放入三级缓存。
填充B的属性时，发现需要注入Bean A。

解决A的早期引用：
从三级缓存中找到A的工厂对象，调用getObject()生成A的早期引用（可能是原始对象或代理对象）。
将A的早期引用放入二级缓存，并从三级缓存移除其工厂对象。
此时Bean B完成属性注入，初始化后放入一级缓存。

完成Bean A的初始化：
Bean A获取到已初始化的Bean B，继续完成属性注入和初始化。
将A从二级缓存移到一级缓存，供后续使用。

2.spring mvc

定义：一种软件设计典范，用一种业务逻辑、数据、界面显示分离的方法组织代码，将业务逻辑聚集到一个部件里面，在改进和个性化定制界面及用户交互的同时，不需要重新编写业务逻辑。

view:为用户提供使用界面，与用户直接进行交互。

model:代表一个存取数据的对象或 JAVA POJO.模型分为两类，一类称为数据承载 Bean，一类称为业务处理Bean

controller: 用于将用户请求转发给相应的 Model 进行处理，并根据 Model 的计算结果向用户提供相应响应

流程：

（1）请求传到前端控制器，调用对应的处理器映射器，处理器映射器根据请求url找到具体的处理器，生成一系列处理器执行链返回给前端控制器

（2）根据处理器找到对应的处理器适配器，然后根据对应的控制器适配器处理一系列的操作

（3）处理器执行，然后将返回的结果ModelAndView,处理器适配器将执行结果传给前端控制器

（4）前端处理器把数据传给modelandview传给视图解析器解析后生成具体view
（5）然后根据view进行渲染

过滤器和拦截器的区别：

过滤器：java servlet的规范，主要是对要进入容器内的请求进行处理

拦截器：spring的一种规范，主要是对访问控制器的请求做出处理

3.spring boot

定义：一种框架

优点：

简化开发：提供一系列的开箱即用的组件和自动配置，简化了项目的配置和开发过程，开发人员可以更专注于业务逻辑的实现，而不需要花费过多时间在繁琐的配置上。

快速启动：Spring Boot提供了快速的应用程序启动方式

自动化配置：Spring Boot通过自动配置功能，根据项目中的依赖关系和约定俗成的规则来配置应用程序，减少了配置的复杂性，使开发者更容易实现应用的最佳实践。

springBoot优于spring：

Spring Boot 提供了自动化配置，大大简化了项目的配置过程。通过约定优于配置的原则，很多常用的配置可以自动完成，开发者可以专注于业务逻辑的实现。

Spring Boot 提供了快速的项目启动器，通过引入不同的 Starter，可以快速集成常用的框架和库

Spring Boot 默认集成了多种内嵌服务器（如Tomcat、Jetty、Undertow），无需额外配置，即可将应用打包成可执行的 JAR 文件，方便部署和运行。
